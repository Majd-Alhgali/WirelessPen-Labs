# خطة فحص محافظة لشبكة `Taim Starlink`

## الملخص
- الهدف: تنفيذ سيناريو عملي عالي الفائدة بعد استبعاد WPS وكسر كلمة Wi‑Fi، مع التركيز أولًا على الراوتر.
- النطاق: شبكة `Taim Starlink` فقط، بعد الاتصال بها فعليًا.
- الشدة: محافظة، بدون brute force، بدون تعطيل خدمات، وبدون إنشاء backdoors.
- المسار الأساسي: اكتشاف الراوتر، تحديد لوحة الإدارة والخدمات، فحص default credentials يدويًا، تقييم firmware، UPnP، PMF/802.11w، والخدمات المكشوفة داخليًا.

## خطوات التنفيذ
1. **تجهيز الأدوات على Kali**
   - تثبيت أدوات الفحص:
     ```bash
     sudo apt update
     sudo apt install -y nmap arp-scan miniupnpc curl whatweb snmp nbtscan avahi-utils dnsutils
     ```
   - الهدف من الأدوات:
     - `nmap`: فحص المنافذ والخدمات.
     - `arp-scan`: اكتشاف الأجهزة داخل الشبكة.
     - `miniupnpc`: اختبار UPnP.
     - `whatweb/curl`: بصمة لوحة الراوتر.
     - `snmp/nbtscan/avahi`: اكتشاف خدمات داخلية إضافية.

2. **تثبيت معلومات الشبكة بعد الاتصال بـ `Taim Starlink`**
   - تحديد عنوان الجهاز والبوابة:
     ```bash
     ip route
     ip -brief addr
     ip neigh show
     ```
   - تحديد SSID/BSSID الحالي:
     ```bash
     iw dev wlan0 link
     nmcli -f active,ssid,bssid,chan,rate,signal,security dev wifi list --rescan no
     ```
   - القاعدة: لا يبدأ أي فحص حتى يكون SSID هو `Taim Starlink` أو BSSID مطابقًا للشبكة المقصودة.

3. **اكتشاف الراوتر ولوحة الإدارة**
   - اعتبار default gateway هو الهدف الأول.
   - فحص منافذ إدارة شائعة بشكل محافظ:
     ```bash
     nmap -sS -sV -O --top-ports 100 <GATEWAY_IP>
     ```
   - فحص منافذ إدارة محددة إذا لم تظهر في الفحص الأول:
     ```bash
     nmap -sV -p 22,23,53,80,443,8080,8443,7547,1900,5000,8291 <GATEWAY_IP>
     ```
   - بصمة الويب:
     ```bash
     curl -I http://<GATEWAY_IP>/
     curl -k -I https://<GATEWAY_IP>/
     whatweb http://<GATEWAY_IP>/
     whatweb https://<GATEWAY_IP>/
     ```

4. **فحص default password وضعف لوحة الراوتر**
   - بعد معرفة الشركة/الموديل من الواجهة أو headers أو صفحة login:
     - البحث عن بيانات الدخول الافتراضية للموديل من مصادر الشركة أو الملصق على الجهاز.
     - تجربة عدد محدود يدويًا فقط مثل:
       - `admin/admin`
       - `admin/password`
       - `admin/<blank>`
       - بيانات الملصق إن وجدت
   - ممنوع في هذه الخطة:
     - brute force.
     - hydra.
     - قوائم كلمات مرور.
     - محاولات كثيرة قد تسبب lockout.
   - معيار النجاح:
     - الدخول بلوحة الراوتر بكلمة افتراضية أو ضعيفة.
     - أو إثبات أن الدخول محمي بكلمة غير افتراضية.

5. **فحص firmware والموديل**
   - من لوحة الإدارة أو صفحة status يتم تسجيل:
     - vendor
     - model
     - hardware version
     - firmware version
   - مقارنة الإصدار مع صفحة الدعم الرسمية للشركة.
   - المخرجات المطلوبة:
     - هل firmware قديم؟
     - هل توجد CVEs معروفة للموديل/الإصدار؟
     - هل التحديث متاح؟
   - لا يتم استغلال CVEs في هذا السيناريو؛ فقط توثيق الخطر وخطوة التحديث.

6. **فحص UPnP والخدمات المكشوفة داخليًا**
   - اختبار UPnP:
     ```bash
     upnpc -l
     ```
   - النتائج المهمة:
     - هل UPnP مفعّل؟
     - هل توجد port mappings غريبة؟
     - هل توجد أجهزة داخلية مكشوفة للإنترنت؟
   - فحص الراوتر داخليًا:
     ```bash
     nmap -sV -sU --top-ports 50 <GATEWAY_IP>
     ```
   - التركيز على:
     - Telnet `23`
     - SSH `22`
     - HTTP/HTTPS
     - TR-069 `7547`
     - UPnP/SSDP `1900`
     - DNS مفتوح داخليًا

7. **فحص PMF / 802.11w**
   - أثناء الاتصال بـ `Taim Starlink`:
     ```bash
     wpa_cli status | grep -Ei 'ssid|bssid|key_mgmt|pmf'
     iw dev wlan0 scan | grep -A40 -i 'SSID: Taim Starlink'
     ```
   - التفسير:
     - `pmf=0`: غير مفعل.
     - `pmf=1`: اختياري.
     - `pmf=2`: مطلوب/إجباري.
   - معيار الفجوة:
     - إذا كان PMF غير مفعل أو اختياري فقط، فالشبكة أكثر قابلية لهجمات deauth، وهذا ينسجم مع ملف الالتقاط السابق الذي احتوى على عدد كبير من Deauthentication frames.

8. **اكتشاف أجهزة IoT لاحقًا بعد الراوتر**
   - فحص محافظ للشبكة:
     ```bash
     sudo arp-scan --localnet
     nmap -sn <SUBNET_CIDR>
     ```
   - ثم فحص خدمات للأجهزة المكتشفة فقط:
     ```bash
     nmap -sV --top-ports 100 <DEVICE_IP>
     ```
   - يتم تصنيف الأجهزة حسب:
     - كاميرات
     - تلفاز/رسيفر
     - هواتف
     - طابعات
     - أجهزة ذكية
   - فحص كلمات المرور الضعيفة يكون يدويًا فقط وعلى واجهات الإدارة المعروفة، بدون brute force.

## معايير النجاح
- تحديد عنوان الراوتر وخدماته المكشوفة داخليًا.
- معرفة هل لوحة الإدارة تستخدم default credentials أو كلمة ضعيفة.
- معرفة حالة firmware وهل يحتاج تحديثًا.
- معرفة هل UPnP مفعّل وهل توجد mappings خطرة.
- معرفة حالة PMF/802.11w.
- إنتاج قائمة فجوات مرتبة حسب الأولوية:
  1. حرجة: default admin password، Telnet مفتوح، UPnP mappings خطرة.
  2. عالية: firmware قديم بثغرات معروفة، TR-069 مكشوف داخليًا.
  3. متوسطة: PMF غير مفعل، خدمات إدارة كثيرة داخليًا.
  4. منخفضة: معلومات banner زائدة أو أسماء أجهزة مكشوفة.

## الافتراضات
- التنفيذ سيكون بعد الاتصال الفعلي بشبكة `Taim Starlink`، وليس الشبكة الحالية `Me`.
- WPS غير مفعل كما ذكرت، لذلك لن يكون ضمن الخطة.
- الفحص محافظ ولا يتضمن brute force أو استغلال فعلي أو backdoors.
- إذا لم توجد لوحة إدارة على البوابة، سيتم البحث عنها عبر فحص الشبكة داخليًا لاكتشاف عنوان الإدارة الصحيح.
