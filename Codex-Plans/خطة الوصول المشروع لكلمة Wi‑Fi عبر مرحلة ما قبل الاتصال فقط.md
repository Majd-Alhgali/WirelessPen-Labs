# خطة الوصول المشروع لكلمة Wi‑Fi عبر مرحلة ما قبل الاتصال فقط

## الملخص
- سنستبعد فحص الراوتر، التصيد، صفحات الدخول المزيفة، WPS، وأي هجوم داخل الشبكة.
- المسار الوحيد الواقعي من النتائج الحالية هو: استغلال غياب PMF لالتقاط handshake نظيف، ثم محاولة استرجاع PSK offline.
- أفضل ملف حالي للاستخدام هو `taim_client_3e_test-01.cap` لأنه يحتوي EAPOL كاملًا: `M1/M2/M3/M4` مع `EAPOL pairs best: 1`.
- الثغرات المكتشفة لا تكشف كلمة السر مباشرة:
  - غياب PMF يسمح بـ deauth/reconnect والتقاط handshake.
  - WPS غير ظاهر، فلا يوجد مسار PIN.
  - WPA/WPA2 mixed موجود، لكن cipher هو `CCMP` فقط، فلا يوجد downgrade مفيد إلى TKIP.

## خطوات التنفيذ
1. **تحويل أفضل handshake إلى صيغة Hashcat**
   - استخدم الملف الأنظف:
     ```bash
     hcxpcapngtool -o taim_3e.22000 taim_client_3e_test-01.cap
     hashcat --identify taim_3e.22000
     ```
   - معيار النجاح: يتعرف Hashcat على `WPA-PBKDF2-PMKID+EAPOL` mode `22000`.

2. **تأكيد جودة المادة قبل أي تخمين**
   - افحص الـ cap:
     ```bash
     aircrack-ng taim_client_3e_test-01.cap
     hcxpcapngtool taim_client_3e_test-01.cap
     ```
   - لا تستخدم `taim_preauth-01.cap` للتخمين لأنه لا يحتوي handshake.
   - استخدم `taim_deauth_test-01.cap` كاحتياطي فقط؛ فيه أكثر من زوج EAPOL لكنه يحتوي deauth أكثر.

3. **بناء قاموس خاص صغير مبني على البيئة**
   - مصادر الكلمات:
     - `taim`, `Taim`, `starlink`, `Starlink`, `TaimStarlink`
     - أسماء العائلة/المكان/المزود إن كانت معروفة لديك
     - أنماط محلية شائعة: سنة، رقم هاتف، اسم + سنة، اسم + رمز
   - ممنوع في هذه الخطة استخدام قوائم عامة ضخمة كبداية؛ الهدف قياس الذكاء لا الحجم.

4. **تشغيل مراحل تخمين منظمة بميزانية واضحة**
   - المرحلة الأولى: كلمات خاصة + rules خفيفة:
     ```bash
     hashcat -m 22000 taim_3e.22000 base.txt -r /usr/share/hashcat/rules/best64.rule
     ```
   - المرحلة الثانية: hybrid واقعي:
     ```bash
     hashcat -m 22000 taim_3e.22000 base.txt ?d?d?d?d
     hashcat -m 22000 taim_3e.22000 base.txt ?s?d?d?d?d
     ```
   - المرحلة الثالثة: masks محددة فقط، لا brute force عشوائي مفتوح:
     ```bash
     hashcat -m 22000 taim_3e.22000 -a 3 Taim?d?d?d?d
     hashcat -m 22000 taim_3e.22000 -a 3 Starlink?d?d?d?d
     hashcat -m 22000 taim_3e.22000 -a 3 Taim?s?d?d?d?d
     ```
   - معيار التوقف: إذا لم تظهر نتيجة ضمن masks منطقية ومحدودة، نسجل أن PSK غالبًا ليس من نمط بشري بسيط.

5. **تحليل النتائج**
   - إذا نجح الاسترجاع:
     - وثّق النمط الذي نجح، لا الكلمة نفسها في التقارير العامة.
     - قيّم سبب الضعف: اسم الشبكة، سنة، رمز شائع، طول قصير، نمط محلي.
   - إذا فشل:
     - النتيجة المهنية: handshake صالح لكن كلمة السر قوية ضد التخمين الحالي.
     - الفجوة تبقى PMF/deauth، لا ضعف PSK.

## الاختبارات والقبول
- قبول مادة الالتقاط:
  - `aircrack-ng` يعرض `WPA (1 handshake)`.
  - `hcxpcapngtool` يعرض `M1/M2/M3/M4`.
  - `hashcat --identify` يتعرف على mode `22000`.
- قبول مسار التخمين:
  - كل مرحلة لها نطاق محدود ومبرر.
  - لا تشغيل brute force كامل غير محدود.
  - لا تصيد ولا Evil Twin لجمع كلمات مرور.
- قبول التقرير:
  - يفرّق بين “قابلية التقاط handshake” و“قابلية كسر PSK”.
  - يذكر أن غياب PMF يساعد على الالتقاط فقط، ولا يكسر كلمة السر وحده.

## الافتراضات
- سنبقى في مرحلة pre-auth فقط.
- لن نستخدم الراوتر أو Gateway IP أو لوحة الإدارة.
- لن نستخدم WPS لأنه غير ظاهر في `wash`.
- كلمة السر إن كانت عشوائية بطول جيد فلن تُسترجع عمليًا من handshake وحده.
- الهدف التعليمي هو بناء منهجية دقيقة، وليس تشغيل قوائم عشوائية ضخمة بلا فرضية.
