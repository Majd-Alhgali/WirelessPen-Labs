تم تنفيذ الخطة كحزمة تشغيل جاهزة داخل:

[evil-twin-lab](/home/majd/Documents/attacks/evil-twin-lab)

ما تم إنشاؤه:

- [hostapd-open.conf](/home/majd/Documents/attacks/evil-twin-lab/configs/hostapd-open.conf): AP مفتوح باسم `Taim Starlink LAB`.
- [hostapd-wpa2.conf](/home/majd/Documents/attacks/evil-twin-lab/configs/hostapd-wpa2.conf): AP تعليمي WPA2 بكلمة اختبارية.
- [dnsmasq.conf](/home/majd/Documents/attacks/evil-twin-lab/configs/dnsmasq.conf): DHCP/DNS معزول على `10.99.0.0/24`.
- [index.html](/home/majd/Documents/attacks/evil-twin-lab/www/index.html): صفحة تعليمية لا تطلب ولا تجمع بيانات.
- [start-open.sh](/home/majd/Documents/attacks/evil-twin-lab/scripts/start-open.sh): تشغيل الوضع المفتوح.
- [start-wpa2.sh](/home/majd/Documents/attacks/evil-twin-lab/scripts/start-wpa2.sh): تشغيل وضع WPA2 التعليمي.
- [stop.sh](/home/majd/Documents/attacks/evil-twin-lab/scripts/stop.sh): إيقاف اللاب واسترجاع NetworkManager.
- [watch-logs.sh](/home/majd/Documents/attacks/evil-twin-lab/scripts/watch-logs.sh): مراقبة logs.

تحققت من:

- وجود الأدوات المطلوبة.
- دعم البطاقة لـ AP mode.
- صحة سكربتات Bash نحويًا.
- صحة إعداد `dnsmasq`.

لم أشغّل الـ AP فعليًا لأن `sudo` يحتاج كلمة مرور عندك. للتشغيل:

```bash
cd /home/majd/Documents/attacks/evil-twin-lab
sudo ./scripts/start-open.sh wlp4s0
```

في نافذة ثانية:

```bash
cd /home/majd/Documents/attacks/evil-twin-lab
./scripts/watch-logs.sh
```

ثم اتصل من هاتفك أو جهاز اختبار بالشبكة:

```text
Taim Starlink LAB
```

وافتح:

```text
http://10.99.0.1:8080/
```

للإيقاف:

```bash
cd /home/majd/Documents/attacks/evil-twin-lab
sudo ./scripts/stop.sh wlp4s0
```

تنبيه عملي: تشغيل `start-open.sh` سيخرج `wlp4s0` من إدارة NetworkManager مؤقتًا، لذلك قد ينقطع اتصال Wi‑Fi الحالي إلى أن تشغّل `stop.sh`.
